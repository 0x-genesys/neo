# Changes - April 30, 2026

## Summary

Fixed curriculum dataset download issue on Kaggle where files were downloaded to HuggingFace cache but not copied to the target directory, causing training to fail.

## Issue

User reported training failure on Kaggle:
```
FileNotFoundError: Source file not found: data/balanced_300m_curriculum/wikitext_train.bin
```

Files were successfully downloaded to HF cache but not accessible in the target directory.

## Changes Made

### 1. Fixed Dataset Downloader (`src/dataset_downloader.py`)

**Problem**: `hf_hub_download()` with `local_dir` parameter was unreliable on Kaggle

**Solution**: 
- Download to HF cache first (reliable)
- Explicitly copy files to target directory
- Verify all files are in place after download

**Code Changes**:
```python
# Before: Unreliable direct download
downloaded_path = hf_hub_download(
    repo_id=repo_id,
    filename=filename,
    local_dir=str(dataset_dir),  # Sometimes doesn't work
    local_dir_use_symlinks=False,
    resume_download=True,
    repo_type="dataset"
)

# After: Reliable download + explicit copy
downloaded_path = hf_hub_download(
    repo_id=repo_id,
    filename=filename,
    resume_download=True,
    repo_type="dataset"
)

# Explicitly copy to target directory
expected_path = dataset_dir / filename
if downloaded_path != str(expected_path):
    import shutil
    shutil.copy2(downloaded_path, expected_path)
```

**Enhanced Verification**:
- Check each file exists in target directory
- Show file sizes
- Clear error messages if files missing
- Separate handling for curriculum vs standard datasets
